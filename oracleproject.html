담당 구현 업무 👌



1️⃣ 요구사항 분석 / ERD 작성

2️⃣ DDL / DummyData 작성

3️⃣ DML(프로시저 / 뷰 / 함수)

요구사항 분석 / ERD 작성

요구사항을 분석한 후 필요한 테이블을 선정하고 관리자/ 교사/ 교사_강의가능과목/ 개설과정/ 과정 테이블의 속성과 관계를 고려하여 ERD를 작성했다.

NEXT

테이블 생성 / 더미데이터 생성

ERD를 기반으로 각 테이블의 CREATE, DROP문을 생성하고 ALTER로 기본키와 외래키, 제약사항을 작성.
시스템 사용 기간을 1년으로 가정하고 INSERT INTO를 사용하여 더미데이터를 생성.
SELECT로 데이터를 확인하며 구현하였다.

NEXT

업무에 필요한 PL/SQL 작성

출결점수를 입력,수정,삭제하는 프로시저를 생성하고, 외래키로 연결되어 있는 여러개의 테이블을 JOIN하여 필요한 조건을 적용시키고 JOIN과정에서 늘어난 속성에서 필요한 속성만 출력할 수 있도록 VIEW를 생성하였다.
DELETE FROM으로 튜플을 삭제하고 UPDATE로 일부 속성값만 수정할 수 있도록 프로시저를 구현하였고 EXECUTE로 프로시저 실행을 확인하였다.
시험테이블의 속성 중 시험파일등록여부인 'a','b'로 작성되어 있는 속성을 값에 따라 '등록(필기)', '등록(실기)', '미등록'으로 변환하여 출력되도록 함수를 작성하였다.

프로젝트 후 🙏


두번째 프로젝트는 데이터베이스만 다루다보니 훨씬 수월하게 진행하였다.
 사용자 요구사항을 분석하는 것이 제일 관건이었던 프로젝트였다. 요구사항을 분석하고 요구에 맞게 출력하기 위해선 테이블의 속성을 정하는 것이 중요했다. 특히 테이블끼리의 관계를 정하는 것이 헷갈려서 관계를 이해하는데 노력했다. 그래야 관계를 맺은 테이블끼리 JOIN하여 원하는 값을 반환할 수 있었다.
  자주 확인해야 하는 Query는 VIEW로 미리 생성해놓으니 훨씬 구현시간이 단축되고 편리했다. 프로시저는 개발자입장에서만 알아볼 수 있게 테이블과 더미데이터 생성해 놓은 것을 사용자도 한눈에 알아볼 수 있게 변환시키도록 구현하므로써 누구든 사용하기에 한층 더 편해졌다. 개발자는 본인뿐만 아니라 모두가 시스템을 사용하는데 있어서 편하고 즐겁게 사용할 수 있도록 해야 한다는 점이 와닿던 프로젝트였다.
